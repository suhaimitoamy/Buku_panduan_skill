Hari 56: Review Minggu Kedelapan & Kuis SQL Lanjutan!

Kuartal 2: Intermediate | Bulan 3: SQL Lanjutan

Selamat datang di akhir minggu kedelapan! Kamu telah menyelesaikan minggu yang sangat produktif dengan fokus pada teknik-teknik SQL lanjutan. Kita telah bergerak dari dasar-dasar SQL seperti SELECT, WHERE, dan GROUP BY ke konsep yang lebih canggih seperti JOIN, subquery, dan Common Table Expressions (CTE).

Seperti biasa di akhir minggu, mari kita luangkan waktu untuk me-review kembali konsep-konsep penting yang sudah kita pelajari dan menguji pemahaman dengan kuis.

Rangkuman Minggu Kedelapan: SQL Lanjutan

Minggu ini kita fokus pada teknik-teknik SQL yang lebih canggih untuk menggabungkan, memfilter, dan memanipulasi data dari beberapa tabel:

  • INNER JOIN (Hari 50): Kita belajar cara menggabungkan baris dari dua tabel berdasarkan kondisi kecocokan menggunakan INNER JOIN. Hanya baris yang memiliki kecocokan di kedua tabel yang akan muncul di hasil.
  • LEFT JOIN & RIGHT JOIN (Hari 51): Kita mempelajari Outer Joins yang memungkinkan kita mempertahankan semua baris dari satu tabel bahkan jika tidak ada kecocokan di tabel lain. LEFT JOIN mempertahankan semua baris dari tabel kiri, sementara RIGHT JOIN mempertahankan semua baris dari tabel kanan.
  • Multi-Table JOIN & Klausa Lain (Hari 52): Kita berlatih menggabungkan tiga tabel atau lebih menggunakan JOIN berantai, dan juga menerapkan klausa WHERE, GROUP BY, dan fungsi agregat pada hasil gabungan.
  • NULL & UNION (Hari 53): Kita membahas cara menangani nilai NULL menggunakan IS NULL, IS NOT NULL, dan fungsi COALESCE. Kita juga belajar cara menggabungkan hasil dari beberapa query terpisah menggunakan UNION dan UNION ALL.
  • Subquery (Hari 54): Kita mempelajari cara menyisipkan query di dalam query lain (subquery) di berbagai bagian seperti SELECT, FROM, dan WHERE. Kita juga membahas operator untuk subquery seperti IN, EXISTS, ANY, dan ALL.
  • Common Table Expressions (CTE) (Hari 55): Kita belajar cara mendefinisikan hasil query sementara menggunakan klausa WITH untuk membuat query kompleks lebih mudah dibaca dan dipelihara. Kita juga menyinggung CTE rekursif untuk query hierarkis.

Setelah minggu kedelapan ini, kamu seharusnya sudah bisa:

  • Menggabungkan data dari beberapa tabel menggunakan berbagai jenis JOIN.
  • Menangani nilai NULL dengan benar dalam query.
  • Menggabungkan hasil dari beberapa query menggunakan UNION.
  • Menulis subquery di berbagai bagian query utama.
  • Menggunakan CTE untuk membuat query kompleks lebih mudah dibaca.

Ini adalah keterampilan SQL lanjutan yang sangat berharga untuk analisis data yang lebih kompleks!

Kuis Santai Minggu Kedelapan

Mari uji pemahaman SQL lanjutanmu! Pilih jawaban yang paling tepat.

1. Jika kamu ingin menampilkan semua pelanggan dan pesanan mereka (jika ada), jenis JOIN yang tepat adalah...

  • a. INNER JOIN
  • b. LEFT JOIN (dengan Pelanggan di sebelah kiri)
  • c. RIGHT JOIN (dengan Pelanggan di sebelah kanan)
  • d. FULL JOIN

2. Untuk menggabungkan tiga tabel (Pelanggan, Pesanan, Produk), kamu perlu...

  • a. Menggunakan TRIPLE JOIN
  • b. Menggunakan dua INNER JOIN berantai
  • c. Menggunakan UNION untuk menggabungkan hasil dari dua JOIN terpisah
  • d. Menggunakan subquery di klausa FROM

3. Untuk mencari pelanggan yang belum pernah melakukan pesanan, kamu bisa menggunakan...

  • a. SELECT * FROM Pelanggan WHERE IDPelanggan = NULL IN (SELECT IDPelanggan FROM Pesanan)
  • b. SELECT * FROM Pelanggan WHERE IDPelanggan NOT IN (SELECT IDPelanggan FROM Pesanan)
  • c. SELECT * FROM Pelanggan LEFT JOIN Pesanan ON Pelanggan.IDPelanggan = Pesanan.IDPelanggan WHERE Pesanan.IDPesanan = NULL
  • d. SELECT * FROM Pelanggan LEFT JOIN Pesanan ON Pelanggan.IDPelanggan = Pesanan.IDPelanggan WHERE Pesanan.IDPesanan IS NULL

4. Untuk menggabungkan daftar kota dari tabel Pelanggan dan Supplier tanpa duplikat, kamu menggunakan...

  • a. SELECT Kota FROM Pelanggan JOIN Supplier ON Pelanggan.Kota = Supplier.Kota
  • b. SELECT Kota FROM Pelanggan UNION SELECT Kota FROM Supplier
  • c. SELECT Kota FROM Pelanggan UNION ALL SELECT Kota FROM Supplier
  • d. SELECT DISTINCT Kota FROM Pelanggan, Supplier

5. Untuk menampilkan produk dengan harga di atas rata-rata menggunakan subquery, sintaks yang benar adalah...

  • a. SELECT * FROM Produk WHERE Harga > AVG(Harga)
  • b. SELECT * FROM Produk WHERE Harga > (SELECT AVG(Harga) FROM Produk)
  • c. SELECT * FROM Produk HAVING Harga > (SELECT AVG(Harga) FROM Produk)
  • d. SELECT * FROM Produk WHERE Harga > AVG(SELECT Harga FROM Produk)

6. Jika kamu ingin menampilkan pelanggan yang memiliki total nilai pesanan lebih dari 1.000.000, pendekatan yang tepat adalah...

  • a. Menggunakan INNER JOIN dan WHERE SUM(Nilai) > 1000000
  • b. Menggunakan INNER JOIN, GROUP BY, dan HAVING SUM(Nilai) > 1000000
  • c. Menggunakan subquery di klausa WHERE
  • d. Menggunakan LEFT JOIN dan WHERE Nilai > 1000000

7. Common Table Expression (CTE) didefinisikan menggunakan klausa...

  • a. DEFINE
  • b. CREATE TEMPORARY
  • c. WITH
  • d. AS

8. Keuntungan menggunakan CTE dibandingkan subquery adalah...

  • a. CTE selalu lebih cepat daripada subquery
  • b. CTE bisa digunakan berulang kali dalam query yang sama
  • c. CTE bisa diakses dari query lain dalam sesi yang sama
  • d. CTE tidak memerlukan kondisi join

Jawaban:

1. b. LEFT JOIN (dengan Pelanggan di sebelah kiri)

2. b. Menggunakan dua INNER JOIN berantai

3. d. SELECT * FROM Pelanggan LEFT JOIN Pesanan ON Pelanggan.IDPelanggan = Pesanan.IDPelanggan WHERE Pesanan.IDPesanan IS NULL

4. b. SELECT Kota FROM Pelanggan UNION SELECT Kota FROM Supplier

5. b. SELECT * FROM Produk WHERE Harga > (SELECT AVG(Harga) FROM Produk)

6. b. Menggunakan INNER JOIN, GROUP BY, dan HAVING SUM(Nilai) > 1000000

7. c. WITH

8. b. CTE bisa digunakan berulang kali dalam query yang sama

Bagaimana hasilnya? Jika kamu mulai merasa nyaman dengan konsep-konsep ini, kamu sudah di jalur yang benar untuk menjadi analis data yang mahir SQL!

Bersiap untuk Minggu Kesembilan!

Minggu depan, kita akan mulai menjelajahi topik baru: Manipulasi Data dengan SQL. Kita akan belajar cara:

  • Menyisipkan data baru ke dalam tabel dengan INSERT.
  • Memperbarui data yang sudah ada dengan UPDATE.
  • Menghapus data dengan DELETE.
  • Membuat dan memodifikasi struktur tabel dengan CREATE TABLE, ALTER TABLE, dan DROP TABLE.

Ini adalah keterampilan penting untuk mengelola database, meskipun sebagai analis data, kamu mungkin lebih sering menggunakan perintah SELECT untuk mengambil dan menganalisis data.

Selamat atas pencapaian delapan minggu ini! Istirahatlah dengan baik dan siapkan diri untuk minggu kesembilan. Sampai jumpa di hari ke-57!
