Hari 60: Review Minggu Kesembilan & Kuis Manipulasi Data SQL!

Kuartal 3: Advanced | Bulan 3: SQL Lanjutan

Selamat datang di akhir minggu kesembilan! Kamu telah menyelesaikan minggu yang sangat produktif dengan fokus pada manipulasi data dan struktur tabel menggunakan SQL. Kita telah bergerak dari perintah SELECT yang sudah kamu kuasai ke perintah-perintah yang memungkinkanmu menambah, memperbarui, menghapus, dan mengelola struktur data.

Seperti biasa di akhir minggu, mari kita luangkan waktu untuk me-review kembali konsep-konsep penting yang sudah kita pelajari dan menguji pemahaman dengan kuis.

Rangkuman Minggu Kesembilan: Manipulasi Data dan Struktur Tabel

Minggu ini kita fokus pada perintah-perintah SQL untuk memanipulasi data dan struktur tabel:

  1. INSERT (Hari 57): Kita belajar cara menambahkan data baru ke dalam tabel menggunakan perintah INSERT INTO. Kita mempelajari berbagai variasi, termasuk menyebutkan kolom secara eksplisit, menambahkan beberapa baris sekaligus, dan menggunakan subquery sebagai sumber data.
  2. UPDATE (Hari 58): Kita mempelajari cara memperbarui data yang sudah ada dalam tabel menggunakan perintah UPDATE. Kita membahas pentingnya klausa WHERE untuk memilih baris yang akan diperbarui, penggunaan ekspresi dan fungsi dalam klausa SET, serta cara memperbarui data berdasarkan data dari tabel lain menggunakan JOIN.
  3. DELETE (Hari 59): Kita belajar cara menghapus data dari tabel menggunakan perintah DELETE FROM. Kita menekankan pentingnya klausa WHERE untuk menghindari penghapusan data yang tidak diinginkan, serta membahas alternatif seperti TRUNCATE TABLE dan konsep "soft delete".
  4. CREATE TABLE dan ALTER TABLE (Hari 60): Kita mempelajari cara membuat tabel baru dengan perintah CREATE TABLE dan memodifikasi struktur tabel yang sudah ada dengan ALTER TABLE. Kita membahas berbagai tipe data, constraint (seperti PRIMARY KEY, FOREIGN KEY, UNIQUE, NOT NULL), serta cara menambahkan, mengubah, dan menghapus kolom dan constraint.

Setelah minggu kesembilan ini, kamu seharusnya sudah bisa:

  • Menambahkan data baru ke dalam tabel.
  • Memperbarui data yang sudah ada.
  • Menghapus data yang tidak diperlukan.
  • Membuat tabel baru dengan struktur yang sesuai.
  • Memodifikasi struktur tabel yang sudah ada.

Ini adalah keterampilan penting untuk mengelola data dan struktur database, meskipun sebagai analis data kamu mungkin lebih sering menggunakan perintah SELECT.

Kuis Santai Minggu Kesembilan

Mari uji pemahaman SQL manipulasi datamu! Pilih jawaban yang paling tepat.

1

Untuk menambahkan satu baris data baru ke tabel Pelanggan dengan kolom IDPelanggan, Nama, dan Email, perintah yang benar adalah...

a. INSERT Pelanggan VALUES (1, 'Ani', 'ani@email.com');
b. INSERT INTO Pelanggan VALUES (1, 'Ani', 'ani@email.com');
c. INSERT INTO Pelanggan (IDPelanggan, Nama, Email) VALUES (1, 'Ani', 'ani@email.com');
d. INSERT INTO Pelanggan SET IDPelanggan = 1, Nama = 'Ani', Email = 'ani@email.com';
2

Jika kamu ingin menambahkan data ke tabel Pesanan berdasarkan hasil query dari tabel PesananSementara, perintah yang tepat adalah...

a. INSERT INTO Pesanan SELECT * FROM PesananSementara;
b. INSERT INTO Pesanan VALUES (SELECT * FROM PesananSementara);
c. INSERT Pesanan FROM PesananSementara;
d. INSERT INTO Pesanan (PesananSementara);
3

Untuk memperbarui kolom Status menjadi 'Aktif' untuk pelanggan dengan ID 5, perintah yang benar adalah...

a. UPDATE Status = 'Aktif' FROM Pelanggan WHERE IDPelanggan = 5;
b. UPDATE Pelanggan SET Status = 'Aktif' WHERE IDPelanggan = 5;
c. UPDATE Pelanggan WHERE IDPelanggan = 5 TO Status = 'Aktif';
d. SET Status = 'Aktif' IN Pelanggan WHERE IDPelanggan = 5;
4

Apa yang terjadi jika kamu menjalankan perintah UPDATE Pelanggan SET Status = 'Aktif'; tanpa klausa WHERE?

a. Perintah akan gagal karena klausa WHERE wajib.
b. Hanya baris pertama yang akan diperbarui.
c. Semua baris dalam tabel Pelanggan akan diperbarui.
d. Tidak ada yang terjadi karena perintah tidak valid.
5

Untuk menghapus semua pesanan dari pelanggan dengan ID 10, perintah yang benar adalah...

a. DELETE Pesanan WHERE IDPelanggan = 10;
b. DELETE FROM Pesanan WHERE IDPelanggan = 10;
c. DELETE * FROM Pesanan WHERE IDPelanggan = 10;
d. REMOVE FROM Pesanan WHERE IDPelanggan = 10;
6

Apa perbedaan utama antara DELETE FROM Tabel dan TRUNCATE TABLE Tabel?

a. DELETE FROM menghapus baris satu per satu, sementara TRUNCATE TABLE menghapus seluruh tabel termasuk strukturnya.
b. DELETE FROM menghapus baris satu per satu, sementara TRUNCATE TABLE menghapus semua baris sekaligus dan lebih efisien.
c. DELETE FROM hanya bisa digunakan dengan klausa WHERE, sementara TRUNCATE TABLE tidak bisa.
d. Tidak ada perbedaan, keduanya melakukan hal yang sama.
7

Untuk membuat tabel baru dengan kolom IDProduk sebagai primary key, perintah yang benar adalah...

a. CREATE TABLE Produk (IDProduk INT KEY, Nama VARCHAR(100));
b. CREATE TABLE Produk (IDProduk INT PRIMARY, Nama VARCHAR(100));
c. CREATE TABLE Produk (IDProduk INT PRIMARY KEY, Nama VARCHAR(100));
d. CREATE TABLE Produk (IDProduk INT UNIQUE NOT NULL, Nama VARCHAR(100));
8

Untuk menambahkan kolom baru Telepon ke tabel Pelanggan yang sudah ada, perintah yang benar adalah...

a. ALTER TABLE Pelanggan ADD Telepon VARCHAR(20);
b. ALTER TABLE Pelanggan CREATE Telepon VARCHAR(20);
c. MODIFY TABLE Pelanggan ADD Telepon VARCHAR(20);
d. UPDATE TABLE Pelanggan ADD COLUMN Telepon VARCHAR(20);

Bersiap untuk Minggu Kesepuluh!

Minggu depan, kita akan mulai menjelajahi topik baru: Fungsi SQL Lanjutan. Kita akan belajar tentang:

  • Fungsi string untuk manipulasi teks.
  • Fungsi tanggal dan waktu untuk bekerja dengan data temporal.
  • Fungsi matematika untuk perhitungan numerik.
  • Fungsi kondisional seperti CASE untuk logika bersyarat.

Ini adalah keterampilan yang akan memperluas kemampuanmu dalam mengolah dan menganalisis data menggunakan SQL.

Selamat atas pencapaian sembilan minggu ini! Istirahatlah dengan baik dan siapkan diri untuk minggu kesepuluh. Sampai jumpa di hari ke-61!

Tips Hari Ini: Gunakan transaksi (BEGIN TRANSACTION) sebelum menjalankan perintah UPDATE/DELETE penting agar bisa rollback jika ada kesalahan.
